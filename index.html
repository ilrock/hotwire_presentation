Three major components

Turbo Frames

  • Useful to decompose your app UI in atomic containers
  • Any links and forms within the frame will only reload the frame
  • Links and forms outside of a frame can reload it by referencing it via data properties
  • Can be lazy-loaded to improve page rendering speed
  • Great way to update a single "section" of your app UI
  • Common use-cases: table filtering and sorting, multi-step forms
üëÄ Example: GitHub Repo üëÄ

Turbo Streams

  • Somewhat similar to turbo frames, but they can be used to update multiple parts of your UI at once
  • They can perform many actions out of the box (replace, update, prepend, append and remove)
  • Common use-cases - remove a record from a table and update the total number of records
  • They work together with broadcasts (coming up) to help with real-time feautures
